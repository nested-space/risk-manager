# Data Model

## Overview

16 tables organised into the following categories:
- **Core entities**: material, project, manufacturing_processes, component, stage, ncrm_library, counterion
- **Risk sub-tables**: manufacturing_process_risk, component_risk, stage_risk
- **Junction tables**: stage_components, stage_ncrms, component_salt
- **Alias sub-tables**: material_alias, ncrm_library_alias, counterion_alias

All primary keys are UUIDs stored as `TEXT`. All timestamps are stored as `DATETIME`
(UTC). Enums are stored as `TEXT` — no CHECK constraints; Python-layer validation only.

---

## SQLite Adaptations (vs PostgreSQL reference)

| PostgreSQL | SQLite (greenfield) |
|------------|---------------------|
| `UUID` PK + `gen_random_uuid()` | `TEXT` PK, default generated by Python `str(uuid.uuid4())` |
| `ARRAY(Text)` aliases column | Removed — replaced by alias sub-tables |
| `JSONB` column | Removed or replaced with sub-table (see below) |
| `ENUM` DB type | `TEXT` via SQLAlchemy `Enum(MyEnum, native_enum=False)` |
| `DateTime(timezone=True)` | `DATETIME` (UTC ISO 8601 strings) |
| `SERIAL` / auto-increment | `INTEGER PRIMARY KEY AUTOINCREMENT` for surrogate PKs |
| `UniqueConstraint` | Supported in SQLite |
| `CheckConstraint` | Supported in SQLite but not used for enum validation |
| `ForeignKey(..., ondelete="CASCADE")` | Requires `PRAGMA foreign_keys = ON` at connection time |
| `ForeignKey(..., ondelete="RESTRICT")` | Supported in SQLite with FK pragma enabled |

## Table Definitions

### `material`

Central entity representing a chemical substance.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PK | UUID (Python-generated) |
| `name` | TEXT | NOT NULL, UNIQUE | Human-readable name |
| `display_name` | TEXT | NOT NULL | Short label for listings; defaults to `name` |
| `interpret_chemically` | INTEGER (BOOLEAN) | NOT NULL | Whether SMILES is interpreted |
| `smiles` | TEXT | UNIQUE, nullable | SMILES notation string (canonical) |
| `created_at` | DATETIME | NOT NULL | UTC creation timestamp |
| `updated_at` | DATETIME | NOT NULL | UTC last-modified timestamp |

**Indexes:** `idx_material_name` (UNIQUE on name), `idx_material_smiles` (UNIQUE on smiles)

**Relationships:**
- `material_alias` → one-to-many (CASCADE delete)
- `project` → one-to-many via `material_id`
- `component` → one-to-many via `material_id`

---

### `material_alias`

One alias per row for a material.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PK | UUID |
| `material_id` | TEXT | FK → material.id CASCADE, NOT NULL | Parent material |
| `alias` | TEXT | NOT NULL | Alias string |
| `created_at` | DATETIME | NOT NULL | UTC creation timestamp |

**Indexes:** `idx_material_alias_material_id` on `material_id`

---

### `project`

A project links a material to a therapy area and contains manufacturing processes.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PK | UUID |
| `name` | TEXT | NOT NULL | Project name |
| `therapy_area` | TEXT | NOT NULL | Enum: TA (see `src/riskmanager_cli/model/enums.py`) |
| `material_id` | TEXT | FK → material.id RESTRICT, NOT NULL | Associated material |
| `created_at` | DATETIME | NOT NULL | UTC creation timestamp |
| `updated_at` | DATETIME | NOT NULL | UTC last-modified timestamp |

**Indexes:** `idx_project_material_id` on `material_id`

**Relationships:**
- `manufacturing_processes` → one-to-many via `project_id`

> Note: `version` column and `statuses`/`interactions` relationships from the
> reference implementation are excluded.

---

### `manufacturing_processes`

A specific route and process number combination within a project.
Renamed from `manufacturing_process` (reference implementation).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PK | UUID |
| `project_id` | TEXT | FK → project.id RESTRICT, NOT NULL | Parent project |
| `route_number` | INTEGER | NOT NULL | Route identifier within the project |
| `process_number` | INTEGER | NOT NULL | Process identifier within the route |
| `created_at` | DATETIME | NOT NULL | UTC creation timestamp |
| `updated_at` | DATETIME | NOT NULL | UTC last-modified timestamp |

**Constraints:**
- `UNIQUE(project_id, route_number, process_number)` — named `uq_project_mfg_process`

**Indexes:** `idx_mfg_process_project` on `project_id`

**Relationships:**
- `manufacturing_process_risk` → one-to-many (CASCADE delete)
- `component` → one-to-many via `process_id`
- `stage` → one-to-many via `process_id`

> Note: `cqas(JSONB)` and `risks(JSONB)` columns from the reference implementation
> are excluded. `risks` is replaced by the `manufacturing_process_risk` sub-table.
> `version` column is excluded.

---

### `manufacturing_process_risk`

Risk assessments at the manufacturing process level.
New table — mirrors the `component_risk` pattern.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PK | UUID |
| `manufacturing_process_id` | TEXT | FK → manufacturing_processes.id CASCADE, NOT NULL | Parent process |
| `risk_type` | TEXT | NOT NULL | Category of risk (e.g., 'Safety', 'Quality', 'Supply') |
| `name` | TEXT | NOT NULL | Short name / title of the risk |
| `description` | TEXT | nullable | Detailed description |
| `current_level` | INTEGER | nullable | Current risk score (1–10 typical) |
| `proposed_mitigation` | TEXT | nullable | Proposed action to reduce risk |
| `mitigated_level` | INTEGER | nullable | Projected risk score after mitigation |
| `created_at` | DATETIME | NOT NULL | UTC creation timestamp |
| `updated_at` | DATETIME | NOT NULL | UTC last-modified timestamp |

**Indexes:** `idx_mfg_process_risk_process_id` on `manufacturing_process_id`

---

### `component`

A material in the context of a manufacturing process. Defines the component's
role in the overall process (not per-stage; see `stage_components` for stage linkage).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PK | UUID |
| `process_id` | TEXT | FK → manufacturing_processes.id RESTRICT, NOT NULL | Parent process |
| `material_id` | TEXT | FK → material.id RESTRICT, NOT NULL | The material |
| `control_strategy_role` | TEXT | nullable | Role label (e.g., 'API', 'CRUDE', 'RSM') |
| `is_isolated` | INTEGER (BOOLEAN) | NOT NULL, DEFAULT 0 | Whether the component is isolated |
| `created_at` | DATETIME | NOT NULL | UTC creation timestamp |
| `updated_at` | DATETIME | NOT NULL | UTC last-modified timestamp |

**Indexes:**
- `idx_component_mfg_process` on `process_id`
- `idx_component_material` on `material_id`

**Relationships:**
- `component_risk` → one-to-many (CASCADE delete)
- `component_salt` → one-to-many (CASCADE delete)
- `stage_components` → one-to-many via `component_id`

> Note: `properties(JSONB)`, `methods(JSONB)`, `specifications(JSONB)`, and `version`
> from the reference implementation are excluded. `methods` and `specifications` are
> noted for future sub-table implementation.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|

| `name` | TEXT | NOT NULL | Stage name |

### `stage_components`

Junction table linking a stage to a component with a directional type.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PK | UUID |
| `stage_id` | TEXT | FK → stage.id CASCADE, NOT NULL | Parent stage |
| `component_id` | TEXT | FK → component.id RESTRICT, NOT NULL | Linked component |
| `component_type` | TEXT | NOT NULL | `'reactant'` or `'product'` |
| `created_at` | DATETIME | NOT NULL | UTC creation timestamp |
| `updated_at` | DATETIME | NOT NULL | UTC last-modified timestamp |

**Constraints:** `UNIQUE(stage_id, component_id)` — named `uq_stage_component`

**Indexes:**
- `idx_stage_components_stage` on `stage_id`
- `idx_stage_components_component` on `component_id`
- `idx_stage_components_type` on `component_type`

**Valid `component_type` values:** `reactant`, `product`
(Validated at the Python layer via Pydantic; no CHECK constraint in DB.)

## SQLite DDL (Reference)

```sql
PRAGMA foreign_keys = ON;
PRAGMA journal_mode = WAL;

CREATE TABLE material (
    id                  TEXT PRIMARY KEY,
    name                TEXT NOT NULL UNIQUE,
    display_name        TEXT NOT NULL,
    interpret_chemically INTEGER NOT NULL DEFAULT 0,
    smiles              TEXT UNIQUE,
    created_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE material_alias (
    id          TEXT PRIMARY KEY,
    material_id TEXT NOT NULL REFERENCES material(id) ON DELETE CASCADE,
    alias       TEXT NOT NULL,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_material_alias_material_id ON material_alias(material_id);

CREATE TABLE project (
    id           TEXT PRIMARY KEY,
    name         TEXT NOT NULL,
    therapy_area TEXT NOT NULL,
    material_id  TEXT NOT NULL REFERENCES material(id) ON DELETE RESTRICT,
    created_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_project_material_id ON project(material_id);

CREATE TABLE manufacturing_processes (
    id             TEXT PRIMARY KEY,
    project_id     TEXT NOT NULL REFERENCES project(id) ON DELETE RESTRICT,
    route_number   INTEGER NOT NULL,
    process_number INTEGER NOT NULL,
    created_at     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uq_project_mfg_process UNIQUE (project_id, route_number, process_number)
);
CREATE INDEX idx_mfg_process_project ON manufacturing_processes(project_id);

CREATE TABLE manufacturing_process_risk (
    id                       TEXT PRIMARY KEY,
    manufacturing_process_id TEXT NOT NULL REFERENCES manufacturing_processes(id) ON DELETE CASCADE,
    risk_type                TEXT NOT NULL,
    name                     TEXT NOT NULL,
    description              TEXT,
    current_level            INTEGER,
    proposed_mitigation      TEXT,
    mitigated_level          INTEGER,
    created_at               DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at               DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_mfg_process_risk_process_id ON manufacturing_process_risk(manufacturing_process_id);

CREATE TABLE component (
    id                    TEXT PRIMARY KEY,
    process_id            TEXT NOT NULL REFERENCES manufacturing_processes(id) ON DELETE RESTRICT,
    material_id           TEXT NOT NULL REFERENCES material(id) ON DELETE RESTRICT,
    control_strategy_role TEXT,
    is_isolated           INTEGER NOT NULL DEFAULT 0,
    created_at            DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at            DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_component_mfg_process ON component(process_id);
CREATE INDEX idx_component_material    ON component(material_id);

CREATE TABLE component_risk (
    id                  TEXT PRIMARY KEY,
    component_id        TEXT NOT NULL REFERENCES component(id) ON DELETE CASCADE,
    risk_type           TEXT NOT NULL,
    name                TEXT NOT NULL,
    description         TEXT,
    current_level       INTEGER,
    proposed_mitigation TEXT,
    mitigated_level     INTEGER,
    created_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_component_risk_component_id ON component_risk(component_id);

CREATE TABLE counterion (
    id                  TEXT PRIMARY KEY,
    name                TEXT NOT NULL UNIQUE,
    display_name        TEXT NOT NULL,
    interpret_chemically INTEGER NOT NULL DEFAULT 0,
    smiles              TEXT UNIQUE,
    created_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE counterion_alias (
    id            TEXT PRIMARY KEY,
    counterion_id TEXT NOT NULL REFERENCES counterion(id) ON DELETE CASCADE,
    alias         TEXT NOT NULL,
    created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_counterion_alias_counterion_id ON counterion_alias(counterion_id);

CREATE TABLE component_salt (
    id               TEXT PRIMARY KEY,
    component_id     TEXT NOT NULL REFERENCES component(id) ON DELETE CASCADE,
    counterion_id    TEXT NOT NULL REFERENCES counterion(id) ON DELETE RESTRICT,
    stoichiometry    REAL,
    is_fully_defined INTEGER,
    created_at       DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at       DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_component_salt_component_id ON component_salt(component_id);

CREATE TABLE stage (
    id         TEXT PRIMARY KEY,
    process_id TEXT NOT NULL REFERENCES manufacturing_processes(id) ON DELETE RESTRICT,
    name       TEXT NOT NULL,
    number     INTEGER NOT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uq_stage_number_per_mfg_process UNIQUE (process_id, number)
);
CREATE INDEX idx_stage_mfg_process ON stage(process_id);

CREATE TABLE stage_risk (
    id                  TEXT PRIMARY KEY,
    stage_id            TEXT NOT NULL REFERENCES stage(id) ON DELETE CASCADE,
    risk_type           TEXT NOT NULL,
    name                TEXT NOT NULL,
    description         TEXT,
    current_level       INTEGER,
    proposed_mitigation TEXT,
    mitigated_level     INTEGER,
    created_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_stage_risk_stage_id ON stage_risk(stage_id);

CREATE TABLE stage_components (
    id             TEXT PRIMARY KEY,
    stage_id       TEXT NOT NULL REFERENCES stage(id) ON DELETE CASCADE,
    component_id   TEXT NOT NULL REFERENCES component(id) ON DELETE RESTRICT,
    component_type TEXT NOT NULL,
    created_at     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uq_stage_component UNIQUE (stage_id, component_id)
);
CREATE INDEX idx_stage_components_stage     ON stage_components(stage_id);
CREATE INDEX idx_stage_components_component ON stage_components(component_id);
CREATE INDEX idx_stage_components_type      ON stage_components(component_type);

CREATE TABLE ncrm_library (
    id                  TEXT PRIMARY KEY,
    display_name        TEXT NOT NULL,
    name                TEXT NOT NULL UNIQUE,
    interpret_chemically INTEGER NOT NULL,
    smiles              TEXT,
    created_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE ncrm_library_alias (
    id              TEXT PRIMARY KEY,
    ncrm_library_id TEXT NOT NULL REFERENCES ncrm_library(id) ON DELETE CASCADE,
    alias           TEXT NOT NULL,
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_ncrm_library_alias_ncrm_id ON ncrm_library_alias(ncrm_library_id);

CREATE TABLE stage_ncrms (
    id         TEXT PRIMARY KEY,
    ncrm_id    TEXT NOT NULL REFERENCES ncrm_library(id) ON DELETE RESTRICT,
    stage_id   TEXT NOT NULL REFERENCES stage(id) ON DELETE RESTRICT,
    role       TEXT NOT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```
